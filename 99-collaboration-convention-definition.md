# 협업 컨벤션 정의(2-3) Commit 메시지 규칙

커밋 메시지는 **라벨 태그 + 변경 요약** 형태로 작성한다.  
(이 단계에서는 이슈 번호를 넣지 않는다)

형식  
- [Feat] 한 줄 요약
- [Bug] 한 줄 요약
- [Refactor] 한 줄 요약
- [Test] 한 줄 요약
- [Question] 한 줄 요약
- [Performance] 한 줄 요약
- [Documentation] 한 줄 요약

예시  
- git commit -m "[Feat] LiDAR 장애물 거리 퍼블리시 노드 추가"
- git commit -m "[Bug] 정지 상태에서 cmd_vel 반복 발행 문제 수정"
- git commit -m "[Refactor] 제어 로직과 센서 처리 모듈 분리"

---

# 협업 컨벤션 정의(2-4) PR 제목 규칙

PR 제목은 **커밋 메시지의 태그를 그대로 사용하되**,  
앞에 **이슈 번호를 포함한 브랜치 표기([라벨/번호])**를 붙인다.

형식  
- [Feat/이슈번호] 한 줄 요약
- [Bug/이슈번호] 한 줄 요약
- [Refactor/이슈번호] 한 줄 요약
- [Test/이슈번호] 한 줄 요약
- [Question/이슈번호] 한 줄 요약
- [Performance/이슈번호] 한 줄 요약
- [Documentation/이슈번호] 한 줄 요약

예시  
- [Feat/9] ROS2에서 LiDAR 기반 장애물 거리 퍼블리시 노드 추가
- [Bug/12] 정지 상태에서도 cmd_vel이 주기적으로 발행되는 문제 수정
- [Refactor/3] 주행 제어 노드에서 센서 처리와 제어 로직 분리
